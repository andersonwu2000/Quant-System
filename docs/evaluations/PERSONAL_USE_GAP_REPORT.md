## 2. 回測結果可信度

### 2.1 已修復的偏差來源

| 問題 | 修復狀態 | 說明 |
|------|:--------:|------|
| **當日收盤價成交** | ✅ 已修復 | `execution_delay=1`: Day N 訊號 → Day N+1 開盤價成交 |
| **滑價模型太樂觀** | ✅ 已修復 | sqrt impact model: `base_bps + impact_coeff × sqrt(qty/adv)` |
| **Kill Switch 未串接** | ✅ 已修復 | 回測每日呼叫 `kill_switch()`，5% 日跌觸發熔斷 |
| **靜默拒單** | ✅ 已修復 | 拒單記錄在 `rejected_log`，反映在回測報告 |
| **零量交易** | ✅ 已修復 | volume == 0 → REJECT |
| **台股整股單位** | ✅ 已修復 | `.TW`/`.TWO` 自動 lot_size=1000 |
| **T+2 交割** | ✅ 已修復 | `settlement_days` 模擬可用現金鎖定 |
| **前填補無限** | ✅ 已修復 | `max_ffill_days=5` 防止停牌股假交易 |
| **股利未入帳** | ✅ 已修復 | `enable_dividends` 依 ex-date 注入現金 |

### 2.2 剩餘已知偏差

| 問題 | 偏差幅度 | 嚴重度 | 說明 |
|------|----------|:------:|------|
| **存活者偏差** | +2~5% | 🟡 中 | Yahoo/FinMind 不含下市公司。FinMind 資料源時會抑制警告，Yahoo 會顯示。需 EODHD（$83/月）或 TEJ 解決 |
| **漲跌幅未限制** | 不定 | 🟢 低 | 台股 ±10% 限制未模擬。極端行情下回測結果不精確 |
| **價格升降單位** | <0.1% | 🟢 低 | 成交價未取整到合法報價 |

### 2.3 回測指標的可靠度

| 指標 | 可靠度 | 原因 |
|------|:------:|------|
| Sharpe Ratio | 🟢 可信 | 次日成交 + sqrt 滑價已大幅修正 |
| Max Drawdown | 🟢 可信 | 下跌計算準確，kill switch 已串接 |
| 年化報酬 | 🟡 略高 | 存活者偏差仍存在，估計高估 2-5% |
| 勝率 | 🟢 可信 | FIFO 配對準確 |
| 交易次數 | 🟢 可信 | 準確反映策略頻率，含拒單記錄 |

### 2.4 品質驗證工具

系統內建回測品質驗證（`src/backtest/validation.py`）：
- **因果性檢查** `check_causality()`: 打亂時間軸 → 結果應不同
- **確定性檢查** `check_determinism()`: 同參數跑兩次 → 結果完全相同
- **穩健性檢查** `check_sensitivity()`: 微調滑價 ±50% → 結果不應崩潰

## 3. 從回測到實際交易的流程

### 3.1 目前的操作流程（半自動）

```
你想用動量策略管理 100 萬台幣的投資組合：

1. 設定投資組合（API 或 Web UI）
2. 系統每月自動跑策略 → 產生目標權重（APScheduler 排程）
3. 收到 Telegram/LINE/Discord 通知：「建議買進台積電 2 張、賣出聯發科 3 張」
4. 打開券商 App → 手動下單
5. 持倉自動記錄在 DB，下次再平衡基於最新持倉計算
```

### 3.2 理想的操作流程（未來）

```
1. 設定：選策略、設定金額、設定再平衡頻率
2. 系統每月自動跑策略 → 產生建議
3. 你確認後一鍵下單（券商 API）
4. 系統自動追蹤：回測建議 vs 實際執行的績效差異
```

## 8. 結論

### 你現在可以做的事

1. **用回測研究策略，結果基本可信**：次日成交、sqrt 滑價、kill switch 已大幅修正偏差
2. **系統每月自動告訴你「該買什麼、賣什麼」**並推播到手機
3. **到券商 App 手動執行**，系統記錄持倉
4. **用 Walk-Forward Analysis 驗證策略穩健性**

### 下一步

券商 API 介接（Shioaji）是唯一阻礙半自動化的功能。完成後操作流程變為：

```
系統排程跑策略 → 建議交易 → 你確認 → 一鍵下單 → 自動追蹤績效
```

### 真正的風險提醒

1. **回測表現好 ≠ 未來能賺錢** — 過去績效不代表未來表現
2. **過度擬合風險** — 調參數調到回測很漂亮，很可能是 overfitting。用 Walk-Forward Analysis 驗證
3. **執行紀律** — 系統說賣你敢不敢賣？人性是最大的風險
4. **存活者偏差** — 年化報酬可能高估 2-5%，心裡打個折
