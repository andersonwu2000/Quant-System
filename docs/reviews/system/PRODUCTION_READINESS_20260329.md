# 系統生產就緒度評估 — 距離「放心依賴」還有多遠？

**日期**：2026-03-29
**提問**：個人和家族投資能放心依賴系統了嗎？
**回答**：**不能。** 還差 3 個必經階段，每個都不能跳過。

---

## 1. 誠實的現狀

### 策略層面：沒有已驗證的 alpha

| 指標 | 數值 | 含義 |
|------|------|------|
| autoresearch 實驗次數 | 75 | 已跑完 |
| L5 OOS 通過 | 3 個因子（4%） | 極低通過率 |
| 最佳策略 IS Sharpe | 0.94 | 看起來可以 |
| 最佳策略 OOS Sharpe | -0.74 | 實際上虧損 |
| 噪音期望 Sharpe (N=15) | 1.4 | **高於觀測值** |
| Paper trading 實績 | 0 天 | 沒有任何 forward 驗證 |

**核心問題**：觀測到的最佳 Sharpe 0.94 低於 N=15 獨立方向的噪音期望 1.4。統計上無法排除「全是噪音」的可能性。

IS 的好成績（+23.8% CAGR）和 OOS 的差成績（-7.4%）之間的巨大落差，說明回測結果不可信賴。唯一有意義的驗證是 forward testing（paper trading + live）。

## 2. 三個必經階段

### 階段 A：Paper Trading 30-90 天（市場開盤後立即開始）

**目的**：驗證策略在 forward time 是否有 alpha。回測不能證明這件事。

**做法**：
- 用 PaperDeployer 部署通過 L5 的 3 個因子
- SimBroker 模式（不需要 CA 憑證）
- 每日記錄 NAV、持倉、交易
- 每日和大盤 / revenue_momentum 比較

**決策標準**：

| 時間點 | 條件 | 動作 |
|--------|------|------|
| 30 天 | 累計虧損 > 10% 或 MDD > 15% | 停止，換因子或重新研究 |
| 30 天 | 沒崩潰 | 繼續到 90 天 |
| 90 天 | 超額報酬 > 0 且 t-stat > 1.0 | 進入階段 B |
| 90 天 | 超額 ≤ 0 | 回到研究階段 |

**為什麼 30 天不夠**：22 個交易日的 Sharpe SE ≈ 0.21。要區分 Sharpe 差 0.3 的兩個策略需要 ~112 天。30 天只能抓「完全崩潰」。

### 階段 B：小額實盤 3-6 個月

**前置條件**：
- 階段 A 通過
- CA 憑證取得
- 通知告警設定完成
- 營運手冊寫好

**做法**：
- 初始資金：總投資金額的 5-10%（例如總目標 1000 萬，先放 50-100 萬）
- 實盤和 paper trading 同時跑，比較差異
- 每日檢查：對帳、kill switch 是否觸發、訂單是否正確成交
- **每週人工審查**：持倉合理性、成本吻合度、是否有異常交易

**決策標準**：

| 指標 | 門檻 | 說明 |
|------|------|------|
| 實盤 vs paper 的 NAV 差異 | < 2% | 執行品質 |
| 滑價 + 衝擊成本 | < 回測假設的 150% | 成本模型準確性 |
| 異常交易次數 | 0 | 任何非預期交易都是 red flag |
| Kill switch 誤觸發 | 0 | 風控參數需要校準 |
| 人工介入次數 | 記錄 | 系統自主度評估 |

### 階段 C：逐步放大

**前置條件**：階段 B 連續 3 個月無重大問題。

**做法**：
- 每月增加 10-20% 的資金，直到達到目標配置
- 持續監控所有階段 B 的指標
- **任何月份出現 MDD > 10% 或異常交易 → 回到階段 B 的資金水位**

## 3. 目前最大的風險

### 3.1 策略風險（最大）

**我們不知道策略是否有 alpha。**

所有的回測驗證（16 項 Validator、DSR、PBO、Permutation test）都是在回答「在歷史數據上，這個策略是否有統計顯著性」。答案是「勉強有，但不確定」（Sharpe 0.94 < 噪音期望 1.4）。

唯一能回答「策略是否真的有 alpha」的是 forward time 的實際績效。**這是 paper trading 不能省略的根本原因。**

如果 paper trading 90 天後超額 ≤ 0 — 系統的基礎設施仍然有價值（可以繼續研究新因子），但**當前的策略不適合實盤部署**。

### 3.2 執行風險

| 風險 | 嚴重度 | 緩解 |
|------|:------:|------|
| Sinopac API 斷線（每日 03:00 維護） | MEDIUM | 重連機制已實作，指數退避 |
| 零股/整股分流 bug | ✅ 已修 | C-01~C-03 修復完成 |
| Kill switch 雙重清倉 | ✅ 已修 | H-02 double-check 修復 |
| 營收數據 40 天延遲遺漏 | ✅ 已修 | evaluate.py 強制，agent 無法繞過 |
| 數據過期未刷新 | HIGH | Phase AD 未做，開盤前需手動更新 |

### 3.3 營運風險

| 風險 | 嚴重度 | 說明 |
|------|:------:|------|
| 沒有營運手冊 | HIGH | 出事時不知道該做什麼 |
| 通知未設定 | HIGH | Kill switch 觸發了但沒人知道 |
| 單人營運 | MEDIUM | 沒有 backup operator |
| 沒有備份策略 | MEDIUM | 如果主策略崩潰，切換到什麼？ |

### 3.4 市場風險（系統無法控制）

- 台股 7 年牛市（2017-2024）的回測結果不代表未來
- 營收動量因子在空頭市場可能失效（FinLab 最佳因子 MDD -41.7%）
- 單一市場、單一資產類別（台股）— 無分散

## 4. 距離「放心依賴」的差距

```
                    現在                    放心依賴
                     ↓                        ↓
[研究完成] → [paper 30 天] → [paper 90 天] → [小額實盤 3 月] → [逐步放大]
    ✅              ❌              ❌              ❌              ❌
```

**最樂觀時程**（假設一切順利）：
- 2026-04：Paper trading 30 天（下週開盤後開始）
- 2026-06：Paper trading 90 天完成
- 2026-06：取得 CA 憑證（可並行）
- 2026-07~09：小額實盤 3 個月
- 2026-10：**最早可以開始「放心依賴」**

**最可能時程**（考慮策略可能不過 paper trading）：
- Paper trading 90 天後發現超額 ≤ 0 → 回到研究 → 再跑 90 天 paper → 再 3 個月實盤
- **2027 Q1 才可能開始放心依賴**

## 6. 誠實的結論

**系統的基礎設施品質很高** — 1700+ 測試、16 項 Validator、Docker 隔離、風控機制。從工程角度看，這是一個認真做的系統。

**但「工程好」不等於「能賺錢」。**

策略層面的事實是：
1. 75 次實驗只有 3 個因子通過 OOS（4%）
2. 最佳 Sharpe 低於噪音期望
3. OOS 報酬 -7.4%
4. 0 天 forward testing

在這個狀態下依賴系統做實際投資 = **賭博**，不是投資。

**Paper trading 不是可選的步驟 — 它是唯一能把系統從「回測看起來不錯」升級到「有理由相信能賺錢」的途徑。**

跳過 paper trading 直接實盤的風險：你可能在 90 天後發現策略不賺錢，但已經虧了真金白銀。Paper trading 90 天的唯一成本是時間和機會成本 — 但它能避免你在不確定的策略上賠錢。

---

## 7. 一句話

**系統可以信賴它「不會出嚴重的工程事故」（kill switch、對帳、風控都到位了）。但不能信賴它「會賺錢」— 這需要 90 天 paper trading + 3 個月小額實盤的真實數據來證明。**
