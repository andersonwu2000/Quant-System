## 待改善 🟡

### 1. 漲跌停限制（台股 ±10%）

台股有每日漲跌幅限制 ±10%。回測中如果價格漲停，實際上**無法買入**（市場上沒有賣單）。目前引擎假設任何價格都能成交。

**影響**：營收動能策略選的股票經常是營收暴增後連續漲停的，回測可能在漲停板買入但實盤買不到。

**修正方案**：
```python
# 在 weights_to_orders 或 SimBroker 中加入：
if daily_return >= 0.095:  # 接近漲停
    skip_buy = True  # 無法買入
if daily_return <= -0.095:  # 接近跌停
    skip_sell = True  # 無法賣出
```

### 2. 成交量限制

回測假設可以買入任意數量。實盤中，你的訂單不能超過日均成交量的一定比例（通常 < 10%），否則會造成巨大市場衝擊。

**影響**：小型股的日均成交量可能只有幾百張，1000 萬的策略可能無法建倉。

**修正方案**：
```python
# 在 weights_to_orders 中加入：
max_shares = avg_daily_volume * 0.10  # 不超過日均量的 10%
actual_shares = min(target_shares, max_shares)
```

### 3. 營收公布延遲模擬

月營收在 T+10 天公布（每月 10 日前）。回測中是否正確模擬了這個延遲？

**檢查項**：
- 策略在 1 月 1 日的 on_bar 中，看到的最新營收應該是去年 11 月的（12 月的還沒公布）
- 如果策略在 1 月 10 日後的 on_bar，才能看到 12 月的營收

**目前狀態**：`revenue_momentum` 策略用的是 FinMind parquet 中的 `date` 欄位，這是營收的「月份」不是「公布日」。需要加入 publication lag。

**修正方案**：
```python
# 在讀取營收時，加入 ~40 天延遲：
# 1 月營收 → 2 月 10 日前公布 → 可用日期 = revenue_date + 40 天
usable_date = revenue_date + pd.DateOffset(days=40)
rev_df = rev_df[rev_df['usable_date'] <= ctx.now()]
```

### 4. 除權息日處理

台股除權息日股價會下跌（除權、除息）。Yahoo Finance 的 `auto_adjust=True` 已經調整了歷史價格，但：
- 調整後價格用於計算報酬是正確的
- 但**實際下單時用的是未調整價格**
- 如果在除權息日持有，需要模擬股利收入

**目前狀態**：`enable_dividends=False`（預設關閉），除息收入未計入。

### 5. 交易時段外的訊號

策略的 `on_bar()` 在回測中是在收盤時呼叫的。但實盤中，營收公布可能在盤後/盤前。如果策略在盤後接收到營收數據，要到隔天才能交易。

### 6. 整張 vs 零股

台股 1 張 = 1000 股。如果策略目標權重算出的股數不是 1000 的倍數，需要向下取整。這會造成實際權重偏離目標。

**目前狀態**：`fractional_shares=True` 在回測中允許零股，但實盤（除非用零股交易）只能整張。

**影響估算**：
- 1000 萬 × 6.67% = 66.7 萬 / 檔
- 台積電 900 元 × 1000 股 = 90 萬 → 只能買 0 張（不夠 1 張！）
- 需要降低 max_holdings 或提高初始資金

### 7. 倖存者偏差

Yahoo Finance 只有現在還上市的股票。已下市的股票（破產、被收購、下市）不在回測中。

**影響**：回測結果過度樂觀 3-8%（文獻估計）。

**修正方案**：用 FinMind 歷史價格（含已下市股票）。但這需要 FinMind 付費方案的額度。

### 8. 多次回測的 selection bias

我們跑了 13 次實驗才找到 revenue_yoy。如果用同一組數據測試了 N 個因子，最好的那個可能只是統計偶然。

**已緩解**：
- Deflated Sharpe Ratio 校正（StrategyValidator 項 #6）
- PBO = 0%（無過擬合）
- t = 16.1（遠超 Harvey 2016 的 t > 3.0 門檻）

但嚴格來說，OOS 2025 H1 = -3.7% 仍然是真正的 OOS 驗證。
