## 2. 回測結果可信度

這是個人使用最重要的問題：**回測說能賺的，真的能賺嗎？**

### 2.1 會讓回測報酬偏高的問題

| 問題 | 偏差幅度 | 嚴重度 | 說明 |
|------|----------|:------:|------|
| **當日收盤價成交** | +1~2% | 🔴 高 | 策略看到收盤價後，在同一根收盤價成交。實際上你看到收盤價時已經收盤了，最快隔天才能交易 |
| **存活者偏差** | +2~5% | 🔴 高 | Yahoo Finance 不含已下市公司，長期回測只看到「活下來的贏家」 |
| **滑價模型太樂觀** | +0.5~1.5% | 🟡 中 | 固定 5bps 滑價，不考慮訂單大小。大單實際滑價可達 20-50bps |
| **成交量限制太寬鬆** | 不定 | 🟡 中 | 預設允許日成交量 10% 的單量，超過整筆被拒但不報錯 |
| **下單取整損失累積** | +0.2~1% | 🟢 低 | 權重轉換為股數時的取整誤差被丟棄，長期累積 |
| **股利入帳時點錯誤** | ~0% | 🟢 低 | 除息日就入帳，實際要 1-2 個月後才收到現金 |

### 2.2 具體案例說明

**當日收盤價成交問題**（影響最大）：

```
實際情況：
  Day 1 收盤 → 策略算出「買 A 股」的訊號
  Day 2 開盤 → 你才能下單，成交價 ≠ Day 1 收盤價

回測的做法：
  Day 1 收盤 → 策略算出訊號 → 立刻在 Day 1 收盤價成交 ✗
```

這意味著你的回測裡**每筆交易都佔了隔夜價差的便宜**。動量策略（追漲）受影響最大，因為隔天開盤通常會跳空高開。

**存活者偏差問題**：

```
回測 2015-2025 台股動量策略：
  Yahoo Finance 資料 → 只有現在還在上市的股票
  → 那些被你的策略選中但後來下市的公司不會出現在回測中
  → 回測報酬高估 2-5%
```

### 2.3 回測指標的可靠度

| 指標 | 可靠度 | 原因 |
|------|:------:|------|
| Sharpe Ratio | 🟡 偏高 | 受收盤價成交 + 滑價樂觀影響 |
| Max Drawdown | 🟢 可信 | 下跌計算較準確 |
| 年化報酬 | 🟡 偏高 | 多報 3-10%，依策略交易頻率而定 |
| 勝率 | 🟡 參考 | FIFO 配對算法在部分成交情境有誤 |
| Sortino Ratio | 🟡 偏高 | 同 Sharpe，短期回測尤其不穩定 |
| 交易次數 | 🟢 可信 | 準確反映策略頻率 |

### 2.4 回測中的隱形地雷

**Kill Switch 從未被呼叫**：

`src/risk/engine.py` 定義了 5% 日跌幅熔斷機制，但 `src/backtest/engine.py` 的主迴圈**從未呼叫 `kill_switch()`**。回測中即使單日虧損超過 5%，系統也不會停止交易。你看到的回測結果是「沒有熔斷保護」的版本。

**靜默拒單**：

當訂單量超過日成交量 10% 時，SimBroker 會拒絕整筆訂單但不報錯。策略以為下單成功，實際上沒有成交。這些「被吃掉的訂單」不會反映在回測結果中。

**前填補（Forward-Fill）遇到停牌/下市**：

股票停牌後，系統用最後收盤價繼續填補最多 5 天。這段期間你的策略可能在一個「不存在的價格」上交易。極端情況下（公司突然下市），回測會在下市價賣出，但現實中你賣不掉。

## 3. 從回測到實際交易的鴻溝

### 3.1 目前的操作流程（全手動）

```
你想用動量策略管理 100 萬台幣的投資組合：

1. 登入 Web 介面
2. 到回測頁面，選「momentum」策略，設定參數，按「執行」
3. 等結果出來，看到目標權重：台積電 20%、聯發科 15%、...
4. 自己打開券商 App
5. 手動算：100 萬 × 20% = 20 萬 → 20 萬 ÷ 台積電股價 → 幾張
6. 手動下單買進
7. 下個月要再平衡？重複步驟 1-6
8. 想知道績效？自己用 Excel 追蹤
```

### 3.2 理想的操作流程（系統應該做到的）

```
1. 設定：選策略、設定金額、設定再平衡頻率（月/週）
2. 系統每月自動跑策略 → 產生目標權重
3. 系統算出需要的交易 → 顯示「建議買賣清單」
4. 你確認後一鍵下單（或自動執行）
5. 系統追蹤持倉、計算績效
6. 風控觸發時推播通知到手機
```

### 3.3 差距對照表

| 步驟 | 回測中 | 實際使用 | 差距 |
|------|:------:|:--------:|------|
| 策略訊號產生 | ✅ 自動 | ❌ 手動跑回測 | 無排程觸發 |
| 目標權重計算 | ✅ 自動 | ❌ 自己看回測結果 | 無 API 直接取得當前建議權重 |
| 差異計算 | ✅ 自動 | ❌ 自己算 | 無「目前持倉 vs 目標」比較功能 |
| 下單 | ✅ SimBroker | ❌ 手動去券商 App | 無券商介接 |
| 成交確認 | ✅ 即時 | ❌ 自己看券商 App | 無成交回報整合 |
| 持倉追蹤 | ✅ 自動更新 | ❌ 重啟就歸零 | Portfolio 未持久化到 DB |
| 績效記錄 | ✅ 完整 NAV 曲線 | ❌ 自己用 Excel | 無跨 session 績效追蹤 |
| 風控保護 | ⚠️ 部分（kill switch 未啟用） | ❌ 無 | 無即時風控 + 通知 |

## 4. 台股特有問題

### 4.1 整股交易單位

**問題**: 系統預設 `lot_size=1`（一股），但台股在 2020 年前為一張 = 1000 股，之後開放盤中零股交易但流動性差異大。

**影響**: 回測用 1 股為單位計算，但你實際可能需要以 1000 股為單位下單。一張台積電 ~70 萬，如果你的投資組合只有 100 萬，根本買不起 20% 的目標權重。

**位置**: `src/domain/models.py:Instrument.lot_size` 預設為 1

### 4.2 T+2 交割

**問題**: 台股 T+2 交割（今天買，後天才扣款），系統完全未模擬。

**影響**: 回測中同一筆現金可以在同一天買了又賣、賣了又買。實際上你的資金有 2 天鎖定期，可用現金比回測中少。

### 4.3 漲跌幅限制

**問題**: 台股有 ±10% 漲跌幅限制，系統未模擬。

**影響**: 回測中股票可以無限跌（前填補價格），但現實中會觸及跌停後無法賣出。極端行情下的回測結果不可信。

### 4.4 價格升降單位

**問題**: 台股價格有分檔升降單位（10 元以下 0.01 元、10-50 元 0.05 元、50-100 元 0.1 元...），系統固定用 0.01。

**影響**: 回測中的成交價在現實中可能不是合法的報價。偏差通常很小（<0.1%），但高頻交易時會累積。

### 4.5 台股資料品質

**問題**: Yahoo Finance 的台股資料已知問題包括：
- 除權息日價格調整偶爾錯誤
- 部分冷門股成交量為 0 但系統仍會交易
- 無下市公司歷史資料

## 5. 日常使用缺失功能

### 5.1 交易通知（影響：高）

**現狀**: 僅 WebSocket 即時推送，必須開著瀏覽器才能收到。

**個人使用需要**: 策略訊號觸發時，發一則 LINE 或 Telegram 通知「動量策略建議本月買進台積電 5 張、賣出聯發科 3 張」。

### 5.2 持倉持久化（影響：高）

**現狀**: `AppState.portfolio` 純記憶體存儲，伺服器重啟 = 所有資料歸零。

**個人使用需要**: 至少把持倉、現金餘額、交易記錄存到 DB，重啟後能復原。

### 5.3 多投資組合（影響：中）

**現狀**: 全系統一個 Portfolio，所有策略共用。

**個人使用需要**: 「我的動量組合」100 萬 +「老婆的價值組合」50 萬，分開追蹤。

### 5.4 策略排程（影響：中）

**現狀**: 無背景任務觸發策略。

**個人使用需要**: 每月第一個交易日自動跑策略、產生建議交易清單。

### 5.5 績效追蹤（影響：中）

**現狀**: 回測有完整 NAV 曲線，但「實際投資」沒有追蹤機制。

**個人使用需要**: 記錄每次實際交易，計算實際 vs 回測的績效差異。

## 6. 個人使用不需要擔心的事

商業運用報告中的許多問題，在個人使用情境下可以忽略：

| 商業報告中的問題 | 個人使用是否需要 | 原因 |
|-----------------|:----------------:|------|
| HTTPS/TLS | ⚠️ 看情況 | 跑在自己電腦上（localhost）不需要；跑在雲端才需要 |
| Load Balancer / HA | ❌ 不需要 | 一個人用不需要高可用 |
| Prometheus / Grafana | ❌ 不需要 | 不需要企業級監控 |
| SAST 安全掃描 | ❌ 不需要 | 不對外開放 |
| CI/CD 自動部署 | ❌ 不需要 | 手動 `docker compose up` 就好 |
| CSRF 保護 | ❌ 不需要 | 不對外開放 |
| DB 連線池調優 | ❌ 不需要 | 1-3 人使用不會有並行問題 |
| 交易審計合規 | ❌ 不需要 | 個人不受金融監管 |
| 藍綠部署 | ❌ 不需要 | 停個幾分鐘升級沒關係 |
| SLA/SLO 監控 | ❌ 不需要 | 不是服務提供者 |

## 7. 修復優先順序與估時

以「個人能實際用來投資」為目標，按影響力排序：

### P0：回測可信度修復（讓你不會被錯誤的回測結果誤導）

| 項目 | 工作內容 | 估計時間 |
|------|----------|----------|
| **次日成交** | 策略訊號在 Day N 產生，Day N+1 開盤價成交 | 2-3 天 |
| **滑價模型升級** | 滑價 = 基礎滑價 + (下單量 / 日均量) × 衝擊係數 | 1-2 天 |
| **啟用 Kill Switch** | 在回測主迴圈中呼叫 `kill_switch()` | 半天 |
| **拒單記錄** | SimBroker 拒絕訂單時記錄並反映在回測報告中 | 1 天 |
| **台股整股單位** | 根據股價自動設定 lot_size（整張 vs 零股） | 1 天 |

**小計: ~1-1.5 週**

### P1：基本可用性（讓你能實際管理投資組合）

| 項目 | 工作內容 | 估計時間 |
|------|----------|----------|
| **Portfolio 持久化** | 持倉 + 現金 + 交易記錄存 DB，啟動時載入 | 2-3 天 |
| **「建議交易」API** | 輸入策略 + 當前持倉 → 輸出建議買賣清單 | 2-3 天 |
| **策略排程** | 背景任務：每月/每週自動跑策略、產生建議 | 2-3 天 |
| **通知整合** | Telegram Bot 或 LINE Notify 推播交易建議 | 1-2 天 |
| **多投資組合** | Portfolio 加 ID，支援多組合 CRUD | 2-3 天 |

**小計: ~2-2.5 週**

### P2：資料品質提升（讓回測結果更貼近現實）

| 項目 | 工作內容 | 估計時間 |
|------|----------|----------|
| **整合 FinMind** | 台股基本面（P/E、P/B、ROE）+ 產業分類 | 2-3 天 |
| **台股漲跌幅限制** | 模擬 ±10% 漲跌停 | 1 天 |
| **T+2 交割模擬** | 追蹤可用現金 vs 總現金 | 1-2 天 |
| **價格升降單位** | 成交價取整到合法報價 | 半天 |
| **成交量 0 檢查** | 成交量為 0 時不允許交易 | 半天 |

**小計: ~1-1.5 週**

### P3：進階功能（讓使用體驗更好）

| 項目 | 工作內容 | 估計時間 |
|------|----------|----------|
| **實際績效追蹤** | 記錄實際交易、計算 vs 回測績效差異 | 2-3 天 |
| **手動交易記錄** | 可手動輸入在券商 App 實際成交的交易 | 1-2 天 |
| **EODHD 整合** | 去存活者偏差 + 全球覆蓋（需付費 ~$83/月） | 2-3 天 |
| **券商 API 介接** | Fugle 或永豐 API（進入半自動交易） | 1-2 週 |

**小計: ~2-3 週**

### 總計

| 階段 | 時間 | 完成後你能做什麼 |
|------|------|-----------------|
| P0 | ~1-1.5 週 | 回測結果可信，不會被虛高的報酬率誤導 |
| P0 + P1 | ~3-4 週 | 系統產生交易建議 → 你到券商 App 手動下單 → 系統追蹤績效 |
| P0-P2 | ~5-6 週 | 台股回測更精確，基本面策略可用 |
| P0-P3 | ~7-9 週 | 接近半自動化投資工作流 |

## 8. 結論

### 你現在可以做的事

1. **用回測研究策略，但心裡打個折**：回測報酬減 3-10% 才是比較接近現實的數字
2. **用 Web 介面看回測結果**，找出在保守估計下仍有正報酬的策略
3. **手動操作**：看回測建議 → 自己到券商 App 下單

### 最小可用版本需要什麼

如果只做 **P0（回測修復）+ P1（基本可用性）**，大約 **3-4 週** 的開發時間，就能達到：

- 回測結果可信（不會被假數字騙）
- 系統每月自動告訴你「該買什麼、賣什麼」
- Telegram/LINE 收到通知
- 你到券商 App 手動執行
- 系統記錄你的持倉和績效

這對個人投資者來說已經是一個**實用的投資輔助工具**。

### 真正的風險提醒

技術差距可以補，但更重要的是：

1. **回測表現好 ≠ 未來能賺錢** — 過去績效不代表未來表現，這是永恆的真理
2. **過度擬合風險** — 如果你調參數調到回測很漂亮，很可能是 overfitting
3. **執行紀律** — 系統說賣你敢不敢賣？人性是最大的風險
4. **資金管理** — 不要把全部身家放進一個策略，系統目前不支援多組合分散
